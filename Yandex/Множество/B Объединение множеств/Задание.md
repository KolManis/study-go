# Объединение множеств

**Ограничение времени:** 1 секунда  
**Ограничение памяти:** 64.0 Мб  

---

## Условие задачи

Даны несколько множеств целых чисел. Требуется найти **количество различных элементов**, которые встречаются **хотя бы в одном** из данных множеств (т.е. размер объединения всех множеств).

---

## Формат входных данных

Первая строка содержит число `n` (`1 ≤ n ≤ 1000`) — количество множеств.

Далее следуют `n` строк, описывающих множества. В `i`-й строке:
- Сначала указано число `kᵢ` (`0 ≤ kᵢ ≤ 1000`) — размер множества
- Затем указано `kᵢ` различных целых чисел (`1 ≤ x ≤ 10⁶`) — элементы множества

---

## Формат выходных данных

Выведите одно число — **размер объединения всех данных множеств** (количество уникальных элементов, которые встречаются хотя бы в одном множестве).

---

## Примеры

### Пример 1
**Ввод**
```
3
3 1 2 3
4 2 4 5 6
2 1 7
```

**Вывод**
```
7
```

**Пояснение:**
- Множество 1: {1, 2, 3}
- Множество 2: {2, 4, 5, 6}
- Множество 3: {1, 7}
- Объединение: {1, 2, 3, 4, 5, 6, 7} → 7 элементов

### Пример 2
**Ввод**
```
4
2 10 20
3 5 10 15
1 100
3 15 20 25
```

**Вывод**
```
6
```

**Пояснение:**
- Множество 1: {10, 20}
- Множество 2: {5, 10, 15}
- Множество 3: {100}
- Множество 4: {15, 20, 25}
- Объединение: {5, 10, 15, 20, 25, 100} → 6 элементов

---

## Ограничения

1. `1 ≤ n ≤ 1000` (количество множеств)
2. `0 ≤ kᵢ ≤ 1000` (размер каждого множества)
3. `1 ≤ x ≤ 10⁶` (значения элементов)
4. Все элементы внутри одного множества гарантированно различны
5. Общее количество элементов ≤ `n × 1000 = 1,000,000`

---

## Решение

### Алгоритмическая сложность:
- **Временная сложность:** O(N × K), где N — количество множеств, K — средний размер множества
- **Пространственная сложность:** O(U), где U — количество уникальных элементов

---

## Оптимизации


1. **Использование `map[int]bool` в Go** — идиоматичный способ реализации множества
2. **Предварительное резервирование памяти** при известном максимальном размере

Данная задача является классическим примером применения операции **объединения множеств** в программировании.
